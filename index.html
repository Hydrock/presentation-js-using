Что такое using?

Ключевое слово для автоматического управления ресурсами

  • Автоматически освобождает ресурсы при выходе из области видимости
  • Работает по аналогии с try-finally
  • Гарантирует выполнение cleanup-кода
  • Избежание утечек ресурсов

Зачем нужен using?

Проблема

  • Утечки памяти
  • Открытые дескрипторы файлов
  • Незакрытые соединения
  • Забытый cleanup-код

Решение

  • Автоматический вызов dispose
  • Детерминированное освобождение
  • Чистый код
  • Меньше ошибок

Пример: файлы (до)

let file;
try {
    file = await fs.open('data.txt', 'r');
    const content = await file.readFile();
    console.log(content);
} finally {
    if (file) {
        await file.close();
    }
}

Много boilerplate-кода

Пример: файлы (с using)

{
    using file = await fs.open('data.txt', 'r');
    const content = await file.readFile();
    console.log(content);
} // file.close() вызывается автоматически!

Чище и безопаснее!

Пример: кастомный ресурс

class DatabaseConnection {
    #connection;
    
    constructor(url) {
        this.#connection = connect(url);
    }
    
    [Symbol.dispose]() {
        this.#connection.close();
        console.log('Connection closed');
    }
}

{
    using db = new DatabaseConnection('postgres://...');
    db.query('SELECT * FROM users');
} // [Symbol.dispose]() вызывается автоматически

await using

Для асинхронного освобождения ресурсов

class AsyncResource {
    [Symbol.asyncDispose]() {
        return this.close(); // возвращает Promise
    }
}

{
    await using resource = new AsyncResource();
    await resource.doSomething();
} // await resource[Symbol.asyncDispose]() автоматически

Протокол Symbol.dispose

Symbol Тип Описание
Symbol.dispose Синхронный Метод для using
Symbol.asyncDispose Асинхронный Метод для await using 
const resource = {
    [Symbol.dispose]() {
        console.log('Cleanup!');
    }
};

using r = resource; // вызывает r[Symbol.dispose]()

Suppress - игнорирование null/undefined

function readFile(path) {
    using file = fs.openSync(path); // может быть null
    
    // Ошибка, если file === null!
    // Решение:
    using file = fs.openSync(path) ?? { [Symbol.dispose]() {} };
}

// Или SuppressInputStream (экспериментально)

Транспиляция в ES5/ES6

Исходный код

{
    using r = getResource();
    r.use();
}

Результат

{
    var r = getResource();
    try {
        r.use();
    } finally {
        r[Symbol.dispose]();
    }
}

DisposableStack

Утилита для группировки ресурсов

using stack = new DisposableStack();
const r1 = stack.use(getResource1());
const r2 = stack.use(getResource2());
const r3 = stack.use(getResource3());

// Все ресурсы освобождаются одной командой
stack.dispose(); // или автоматически при выходе из scope

// Асинхронная версия:
using asyncStack = new AsyncDisposableStack();
await asyncStack.use(getAsyncResource());

Лучшие практики

  • Используйте using для ресурсов с детерминированным освобождением
  • Реализуйте Symbol.dispose в своих классах
  • Используйте await using для асинхронных ресурсов
  • Группируйте ресурсы с DisposableStack
  • Проверяйте поддержку в целевой среде
