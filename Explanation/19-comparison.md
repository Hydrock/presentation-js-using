# Слайд 19: Сравнение: using vs try-finally

## Описание слайда

Сравнительный анализ двух подходов к управлению ресурсами.

## Таблица сравнения

| Критерий | using | try-finally |
|----------|-------|-------------|
| Читаемость | ✅ Высокая | ⚠️ Средняя |
| Boilerplate | ✅ Минимум | ❌ Много |
| Гарантия очистки | ✅ Автоматическая | ⚠️ Ручная |
| Поддержка | ⚠️ Новая | ✅ Везде |

## Детальное сравнение

### Читаемость

**using**: Код лаконичен и сразу показывает намерение программиста — этот ресурс должен быть освобождён.

```javascript
using file = await fs.open('data.txt', 'r');
// бизнес-логика
```

**try-finally**: Код разбавлен инфраструктурными деталями.

```javascript
let file;
try {
    file = await fs.open('data.txt', 'r');
    // бизнес-логика
} finally {
    if (file) await file.close();
}
```

### Boilerplate

**using**: Минимум кода — только объявление с `using`.

**try-finally**: Много дополнительного кода:
- Объявление переменной вне try
- Блок try
- Блок finally
- Проверка на null/undefined
- Явный вызов очистки

### Гарантия очистки

**using**: Автоматическая гарантия. Невозможно забыть вызвать dispose.

**try-finally**: Зависит от программиста. Легко:
- Забыть проверить на null
- Пропустить await
- Забыть вызвать close

### Поддержка

**using**: Новая фича, требует современных браузеров/Node.js или транспиляции.

**try-finally**: Поддерживается везде с незапамятных времён.

## Когда использовать using

1. **Целевая среда поддерживает** — современные браузеры или Node.js 20.10+
2. **Возможна транспиляция** — проект использует TypeScript или Babel
3. **Множество ресурсов** — значительное сокращение boilerplate
4. **Новый код** — нет необходимости сохранять совместимость

## Когда использовать try-finally

1. **Старые среды** — нельзя транспилировать
2. **Критичный production** — консервативный подход
3. **Короткоживущий код** — скрипты, которые не будут развиваться
4. **Образовательный контент** — для демонстрации базовых концепций

## Производительность

`using` не добавляет накладных расходов по сравнению с try-finally — это просто синтаксический сахар. Транспилированный код идентичен ручному try-finally.

## Миграция

Существующий код можно легко переписать:

```javascript
// До
let resource;
try {
    resource = getResource();
    use(resource);
} finally {
    if (resource) resource[Symbol.dispose]();
}

// После
{
    using resource = getResource();
    use(resource);
}
```

## Ключевой вывод

`using` предпочтительнее для нового кода в поддерживаемых средах. try-finally остаётся запасным вариантом для совместимости.
