# Слайд 3: Что такое using?

## Описание слайда

Введение в концепцию ключевого слова `using` как механизма автоматического управления ресурсами.

## Определение

`using` — это ключевое слово для **автоматического управления ресурсами** в JavaScript, которое гарантирует освобождение ресурсов при выходе из области видимости.

## Ключевые характеристики

### Автоматическое освобождение ресурсов
Ресурсы автоматически освобождаются при выходе из блока кода, в котором они были объявлены. Это происходит детерминированно, в отличие от сборки мусора, которая недетерминирована.

### Аналогия с try-finally
`using` работает по аналогии с конструкцией try-finally, но делает код чище и безопаснее. Программисту не нужно вручную писать блок finally для очистки.

### Гарантия выполнения cleanup-кода
Cleanup-код (код очистки) выполняется гарантированно, даже если:
- Произошло исключение
- Выполнен ранний return
- Блок завершён нормально

### Избежание утечек ресурсов
Автоматическое управление исключает человеческий фактор, когда разработчик может забыть освободить ресурс.

## Принцип работы

```javascript
{
    using resource = getResource();
    // работа с ресурсом
} // здесь автоматически вызывается resource[Symbol.dispose]()
```

## Преимущества перед ручным управлением

| Аспект | Ручное управление | С использованием using |
|--------|-------------------|------------------------|
| Надёжность | Зависит от программиста | Гарантировано |
| Читаемость | Много boilerplate | Минимум кода |
| Поддержка | Склонна к ошибкам | Автоматическая |

## Что должен запомнить слушатель

`using` — это синтаксический сахар над try-finally, который автоматически вызывает метод dispose ресурса при выходе из области видимости.
