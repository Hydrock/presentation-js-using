# Слайд 6: Зачем нужен using?

## Описание слайда

Постановка проблемы и демонстрация того, как `using` её решает.

## Проблемы ручного управления ресурсами

### Утечки памяти
Когда объекты не освобождаются должным образом, память не возвращается системе. В долгоживущих приложениях это может привести к исчерпанию памяти.

Проблема усугубляется тем, что JavaScript имеет автоматическую сборку мусора, но она не работает для "внешних" ресурсов (файлы, соединения, дескрипторы).

### Открытые дескрипторы файлов
Каждая операционная система имеет лимит на количество открытых файлов. Если не закрывать файлы после использования, можно достичь этого лимита, и приложение перестанет работать.

### Незакрытые соединения
Соединения с базой данных, WebSocket, HTTP-соединения — все они потребляют ресурсы. Незакрытые соединения могут:
- Привести к исчерпанию пула соединений
- Оставить транзакции незавершёнными
- Увеличить нагрузку на сервер

### Забытый cleanup-код
Человеческий фактор — разработчик может просто забыть вызвать метод закрытия, особенно в сложных сценариях с множеством ветвлений и обработкой ошибок.

## Решение с использованием using

### Автоматический вызов dispose
`using` автоматически вызывает метод `[Symbol.dispose]()` при выходе из области видимости, что гарантирует очистку.

### Детерминированное освобождение
В отличие от сборки мусора, освобождение происходит в предсказуемый момент — при выходе из блока. Это позволяет точно контролировать, когда ресурс будет освобождён.

### Чистый код
Код становится чище и понятнее:
- Меньше boilerplate
- Явное выражение намерений
- Меньше мест для ошибок

### Меньше ошибок
Автоматизация исключает целый класс ошибок, связанных с неправильным управлением ресурсами.

## До и после

### До (проблемный код)
```javascript
let connection;
try {
    connection = await createConnection();
    await connection.query('...');
    // Что если здесь исключение?
    // А что если return?
} finally {
    if (connection) {  // А если connection не был создан?
        await connection.close();
    }
}
```

### После (с using)
```javascript
{
    using connection = await createConnection();
    await connection.query('...');
} // Всё автоматически!
```

## Ключевой вывод

`using` решает реальные проблемы управления ресурсами, делая код безопаснее и чище.
