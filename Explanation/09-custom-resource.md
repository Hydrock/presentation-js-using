# Слайд 9: Пример: кастомный ресурс

## Описание слайда

Демонстрация создания собственного класса с поддержкой протокола dispose.

## Пример кода

```javascript
class DatabaseConnection {
    #connection;
    
    constructor(url) {
        this.#connection = connect(url);
    }
    
    [Symbol.dispose]() {
        this.#connection.close();
        console.log('Connection closed');
    }
}

{
    using db = new DatabaseConnection('postgres://...');
    db.query('SELECT * FROM users');
} // [Symbol.dispose]() вызывается автоматически
```

## Разбор реализации

### Приватное поле
```javascript
#connection;
```
Использование приватного поля (#) для хранения внутреннего состояния соединения.

### Конструктор
```javascript
constructor(url) {
    this.#connection = connect(url);
}
```
Инициализация ресурса при создании объекта.

### Метод Symbol.dispose
```javascript
[Symbol.dispose]() {
    this.#connection.close();
    console.log('Connection closed');
}
```

Это ключевой момент. Класс реализует метод с ключом `Symbol.dispose`. Этот метод будет автоматически вызван при выходе из области видимости.

Вычисляемое имя свойства `[Symbol.dispose]` означает, что метод доступен через символ, а не через строковое имя.

## Использование

```javascript
{
    using db = new DatabaseConnection('postgres://...');
    db.query('SELECT * FROM users');
} // [Symbol.dispose]() вызывается автоматически
```

При выходе из блока автоматически вызывается `db[Symbol.dispose]()`, который закрывает соединение.

## Преимущества подхода

### Инкапсуляция
Логика освобождения ресурса инкапсулирована внутри класса. Пользователю класса не нужно знать, как именно освобождать ресурс.

### Безопасность
Пользователь не может случайно забыть закрыть соединение — это произойдёт автоматически.

### Гибкость
Класс может содержать сложную логику освобождения:
- Закрытие связанных ресурсов
- Откат транзакций
- Логирование
- Освобождение нескольких ресурсов

## Проверка поддержки

Можно проверить, поддерживает ли объект протокол dispose:

```javascript
if (Symbol.dispose in resource) {
    using r = resource;
} else {
    // ручное управление
}
```

## Практические применения

Этот паттерн полезен для:
- Соединений с базой данных
- Сетевых соединений
- Файловых дескрипторов
- Мьютексов и блокировок
- Любых ресурсов, требующих явного освобождения

## Ключевой вывод

Любой класс может быть адаптирован для работы с `using` путём реализации метода `[Symbol.dispose]()`.
