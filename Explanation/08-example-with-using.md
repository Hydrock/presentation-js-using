# Слайд 8: Пример: файлы (с using)

## Описание слайда

Демонстрация упрощения кода при использовании ключевого слова `using`.

## Пример кода

```javascript
{
    using file = await fs.open('data.txt', 'r');
    const content = await file.readFile();
    console.log(content);
} // file.close() вызывается автоматически!
```

## Разбор кода

### Блок области видимости
```javascript
{
    // ...
}
```
Обратите внимание на блок кода (фигурные скобки). `using` требует наличия области видимости — переменная существует только внутри этого блока.

### Объявление с using
```javascript
using file = await fs.open('data.txt', 'r');
```
Ключевое слово `using` объявляет переменную, которая будет автоматически освобождена при выходе из блока.

### Работа с файлом
```javascript
const content = await file.readFile();
console.log(content);
```
Бизнес-логика работы с файлом — чистая и понятная.

### Автоматическое закрытие
```javascript
} // file.close() вызывается автоматически!
```
При достижении закрывающей скобки автоматически вызывается метод `[Symbol.dispose]()`, который в случае файла закрывает его.

## Преимущества

### Чище и безопаснее
Код стал значительно короче и понятнее. Вся инфраструктура управления ресурсами скрыта за ключевым словом `using`.

### Гарантия выполнения
Cleanup-код выполняется гарантированно, даже если:
- Произошло исключение в блоке
- Выполнен ранний return
- Блок завершён успешно

### Меньше возможностей для ошибки
Не нужно:
- Объявлять переменную вне блока
- Писать try-finally
- Проверять на null
- Помнить о вызове close

## Сравнение строк кода

| Подход | Строк кода |
|--------|------------|
| try-finally | 8 |
| using | 4 |

Уменьшение кода в 2 раза!

## Несколько ресурсов

С `using` работа с несколькими ресурсами становится тривиальной:

```javascript
{
    using file1 = await fs.open('file1.txt', 'r');
    using file2 = await fs.open('file2.txt', 'r');
    // работа с файлами
} // оба файла закрываются автоматически
```

## Ключевой вывод

`using` делает код чище, безопаснее и проще в поддержке.
